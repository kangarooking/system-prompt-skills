# System Prompt Skills

A ready-to-use AI skill pack that distills 15 reusable system prompt design patterns from 165 system prompts used by leading AI products.

This is not a leaked prompt archive or a prompt template dump. The original prompts are the source corpus; this repository preserves transferable methodology: identity, personality, tools, safety, memory, context, search, citation, output control, mobile, voice, and coding-agent system design.

## What These Skills Help With

- Designing stable AI product system prompts without guessing whether to start from persona, tools, safety, or output rules
- Upgrading chat assistants into agents with tool permissions, routing, delegation, and context management
- Reducing prompt injection, unsafe tool use, memory contamination, and unauditable citations
- Building scenario-specific prompts for mobile, voice, coding agents, and AI search products
- Learning common system-prompt patterns across OpenAI, Anthropic, Google, xAI, Perplexity, and other AI products

## What Was Distilled

The pack contains 15 skills extracted from the `system_prompts_leaks/` corpus and organized into four layers.

### Core Architecture

- `persona-design`: identity and role boundary design
- `personality-system`: pluggable personality and tone layers
- `tool-specification`: tool definition, discovery, permissioning, and lifecycle
- `safety-guardrails`: safety defenses, refusal strategy, and ethical boundaries
- `memory-system`: long-term memory, silent application, and sensitive boundaries

### Interaction Control

- `output-formatting`: response length, format, style, and anti-slop rules
- `conversation-flow`: intent classification, clarification, routing, and autonomy control
- `search-integration`: search triggers, query generation, source priority, and retrieval strategy
- `citation-system`: citation formats, attribution, and auditable responses

### Engineering Support

- `context-management`: token budgets, compression, lazy loading, and cache-aware scheduling
- `agent-delegation`: subagent specialization, context isolation, and task lifecycle
- `injection-defense`: prompt injection defense, trust levels, and anti-leakage rules

### Scenario Adaptation

- `voice-optimization`: concise, spoken, audio-safe voice assistant behavior
- `mobile-adaptation`: small-screen response tiers, mobile format limits, and native tool adaptation
- `code-engineering`: coding-agent Git safety, editing workflow, and verification loops

## How to Use

1. Start with `INDEX.md` and choose a composition path based on the product type
2. For a new AI product, start with `persona-design`, `safety-guardrails`, and `output-formatting`
3. For agents, combine `tool-specification`, `conversation-flow`, `agent-delegation`, and `injection-defense`
4. For AI search or RAG, combine `search-integration`, `citation-system`, and `context-management`
5. Integrate the relevant `SKILL.md` trigger conditions, procedures, and boundaries into your agent framework
